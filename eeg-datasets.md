# EEG–音频跨模态解码中的 EEG 离散化与时间语义对齐研究报告

## 执行摘要

围绕“连续 EEG 的离散化（tokenization）+ 与音频的时间/语义对齐”这一目标，我优先筛选了来自 OpenNeuro、Zenodo 以及来自 KU Leuven、Technical University of Denmark 等机构公开发布的 10 个高价值 EEG–音频数据集。若以**连续语音的细粒度语义对齐**为首要目标，最值得优先投入的是  **ds004718 LPPHK** 、**ds005345 LPP Multi-talker** 和  **ds004408** ：它们提供了较长时长的自然语音、较好的公开注释基础，而且前两者的语言学/韵律注释最适合做“多分辨率时间对齐 + 语义对齐”的统一基准。若以**低层时序跟踪与选择性注意**为主，则  **ds006434-exp1Diotic** 、 **KUL AAD** 、**DTU AAD** 与 **4518754 Ultra-high-density 255ch** 更强，因为它们触发明确、音频刺激可追溯、试次结构规则、适合测试 envelope/onset/TRF/CCA/attention decoder 等低层耦合模型。若以**受控音位级对齐**为主，则 **ds006104** 是最好用的“受控小词表/音位”基准，但它并不是自然连续语音，因此更适合作为对齐/离散化模块的受控验证集，而不是最终生态学测试集。音乐方向上，**OpenMIIR** 最适合节拍/拍点/结构级对齐，**MUSIN-G** 适合做高密度自然音乐听觉表征学习，但其对“精确符号级标注与语义注释”的支持明显弱于语音数据集。

从工程角度看，核心矛盾不是“EEG 能否和音频对齐”，而是**采样率、带宽、时延、语义密度**这四种失配同时存在。最稳健的方案不是直接把原始 16 kHz/44.1 kHz 音频 token 与原始 EEG 样本一一对齐，而是采用 **分层多速率表示** ：底层使用 envelope / onset / low-level spectrotemporal 特征对齐 25–100 Hz 的 EEG token 序列；中层使用音素/词边界或 beat/downbeat 对齐 2–20 Hz 的事件 token；高层使用词、句、说话人或乐句级语义 token 对齐更慢的段级 EEG 表征。数据集选择也应按这三层来分工，而不是试图用一个数据集同时解决全部问题。

我不建议把所有公开数据直接混成“一个大池子”来训练深网。至少 **KUL** 数据集官方已经明确提醒了 **trial fingerprint** 与 **gaze bias** 两类严重验证偏差；如果继续使用随机切片交叉验证，尤其是对“直接 EEG 分类 attended speaker / attended side”的方法，结果往往会显著高估。相反，应该采用 **leave-one-trial-out / leave-one-story-out / leave-one-subject-out** 这类严格切分，并明确区分“需要 stimulus side information 的解码”和“纯 EEG 端分类”。

## 数据集逐项可行性

* **ds004408** ：**连续 EEG tokenization 适合度：高；细粒度音频对齐：中高。** 优点是 128 通道、512 Hz、受试者数与总录音量都足够，且是典型的连续有声书听觉范式；非常适合作为“自然连续语音 EEG tokenizer”的主训练集。局限是已抓取官方元数据里并未像 LPPHK 那样直接给出词边界/10 ms prosody 级注释，因此如果你要做词级或音素级监督，通常需要额外 forced alignment。**致命缺陷**没有，但它更偏“连续听觉跟踪”而不是“现成词级对齐基准”。
* **ds005345 LPP Multi-talker** ：**连续 tokenization：高；时间/语义对齐：很高。** 这是我最推荐的开放自然语音数据之一，因为它同时覆盖单讲者与多讲者条件，并且官方仓库里就有 `stimuli/`、`annotation/` 与 quiz 结构；论文还明确强调了详细转写与韵律/语言学注释，非常适合做“低层声学 + 高层语义”的分层对齐。局限是当前仓库 README 与论文摘要在受试者数上存在 25/26 的轻微不一致；另外 EEG 试次级元数据需要你下载 BIDS 后进一步核对。**致命缺陷**没有。
* **ds004718 LPPHK** ：**连续 tokenization：很高；时间/语义对齐：极高。** 若你的目标是“词边界可追溯 + prosody + NLP predictor 一体化”，它几乎是当前公开 EEG 语音数据里的首选之一：52 名受试者、64 通道、1 kHz、20 分钟自然语音、4 sections、词边界、每 10 ms 的 f0/RMS、POS 与词频全部具备。局限是论文已经明确指出，词切分部分依赖人工分词，存在一致性瓶颈，而且 4473 个词里有 707 个在参考语料中频率为 0，这会影响某些 NLP predictor 的稳定性。**致命缺陷**没有，但如果你做跨语种模型，粤语的语言学特性会让英文语音预训练表示的迁移更难。
* **ds006104** ：**连续 tokenization：中；细粒度音位对齐：高。** 这是“受控音位/词级对齐”的优质数据，而不是“自然连续语音”数据。优点是任务标签清楚、事件时间戳明确、受试者来自两个独立研究，适合验证你的离散器是否真学到了 articulatory/phonemic structure。局限在于它不是自然长程语音，且伴随 TMS，会引入 onset separability 与 TMS-target confound；最近工作已明确提醒，在严格 LOSO 与 confound control 下，任务远没有表面上那么容易。**致命缺陷**是：如果你的主目标是长时程自然 EEG tokenization，它只能做模块验证，不能替代自然语音预训练集。
* **ds006434 / exp1Diotic** ：**连续 tokenization：中高；低层时间对齐：极高；语义对齐：低。** 这个数据集对“精确同步工程”特别重要：exp1 使用 64 s epoch、120 trials、25 kHz 原始采样，配套 glottal pulse times、刺激文件、resampling-based clock-drift correction、以及对长静音段的显式处理。它非常适合你研究“EEG–audio alignment 的同步误差、亚皮层/皮层多速率建模、onset/ANM regressor 对齐”。局限是皮层通道只有 32 个，刺激还是为提高 ABR SNR 做过特定构造，语义层标签远不如 LPP 系列丰富。**致命缺陷**是：如果你的目标是词级/句级语义对齐，它并不合适。
* **KUL AAD** ：**连续 tokenization：中高；低层时间对齐：高；语义对齐：低。** 作为 AAD 经典基准，它有足够长的连续双讲者故事、64 通道、WAV 刺激、明确 trial 结构，非常适合 envelope/onset/TRF/CCA/attention decoder。官方仓库还给出了预处理脚本与原始/公开保存版差别。局限极其重要：作者在 2024 年亲自加注警告，指出**trial fingerprint** 和 **gaze bias** 会让很多深度学习结果虚高。**致命缺陷**是：如果你做“纯 EEG 直接分类 attended side/speaker”，但没有 leave-one-trial/story 的严格划分、也没有显式使用 stimulus-correlation 范式，那么这个数据集会误导你。
* **DTU AAD** ：**连续 tokenization：中；低层时间对齐：高；语义对齐：低。** 其优势在于官方直接公开了 `AUDIO.zip`、`EEG.zip`、`DATA_preproc.zip` 和对齐脚本，条件设计上还有不同房间混响强度，非常适合做鲁棒对齐与跨房间泛化。局限是语言学标注不丰富，主要适合声学跟踪与注意选择，而不是 transcript-level semantic alignment。**致命缺陷**是：如果你要做语义 token 对齐或 transcript supervision，它的标签密度不够。
* **Ultra high-density 255ch AAD** ：**连续 tokenization：高；低层时间对齐：高；语义对齐：低。** 255 通道、1 kHz、44.1 kHz WAV 原始刺激，使它成为高空间分辨率 EEG tokenizer 的强基准，非常适合研究 codebook 是否利用了空间拓扑与高密度冗余。局限是只有 4 个长 trial、全部为男性受试者与男性讲者故事，语义监督也很薄。**致命缺陷**是：如果你需要丰富的文本标签、说话人多样性或跨性别泛化，这个数据集不够。
* **OpenMIIR** ：**连续 tokenization：中；音乐时间对齐：高；语义对齐：低。** 它最适合做音乐中的 beat/downbeat/tempo/meter 对齐，以及比较“有歌词/无歌词/纯器乐”三种结构条件；而且是 public-domain，代码和实验复现实例丰富。局限是受试者只有 10 人，片段长度 7–16 s，且作者自己在后续综述中承认意象条件同步差、可变长度 trial 会给表示学习带来困难。**致命缺陷**是：如果你要做稳定的 imagination alignment，不能直接把 perception 条件的方法照搬过去；应优先从 cue-click perception 条件开始。
* **MUSIN-G** ：**连续 tokenization：中高；音乐时间对齐：中；语义对齐：低。** 128 通道自然音乐听觉数据对学习稳定 EEG code 很有价值，尤其适合做“长音乐片段 → EEG token 序列”的自监督建模，并利用熟悉度/愉悦度作为弱监督标签。局限是当前公开摘要没有像 OpenMIIR 那样给出 beat/downbeat、乐句或歌词级注释；而且 full audio 是否完整镜像到仓库、以及采样率/版权状态，最好下载前逐个核对。**致命缺陷**是：如果 full songs 不可直接重分发，或者缺少可复用时间标注，那么它不适合作为“精确音乐–EEG 对齐”的唯一主数据集。

## 方法学与工程方案

从模型设计上，我建议把问题拆成三个同步但不同速率的子任务： **低层声学对齐、事件级对齐、高层语义对齐** 。对于语音，低层主要是 envelope、onset envelope、log-Mel、pitch、voicing、speaker-change 等；事件级主要是 phoneme/word onset、pause、prosodic boundary；高层则是词、句、说话人、段落语义。对于音乐，低层是 broadband envelope、sub-band envelope、chroma、harmonic/percussive decomposition；事件级是 beat/downbeat/onset；高层是乐句、主题、流派、歌词段落。像 ds004718 与 ds005345 这种带转写和 prosody 的数据，天然适合完整三级；KUL/DTU/4518754 更适合第一层；OpenMIIR 更适合第一、二层。

在 EEG 预处理上，不建议一条管线打天下。更合理的是 **双分支 EEG 前端** ：
其一是 **皮层低频分支** ，用于语音包络、音素、词级加工，典型参数为：高通 0.3–0.5 Hz、低通 30–45 Hz、notch 50/60 Hz、重参考（平均参考或 mastoid）后重采样到 128/256 Hz；
其二是 **高频/脑干分支** ，仅针对 ds006434 这类高采样数据保留 1–10 kHz 或更高原始采样、30–2000 Hz 带通，再按任务做 1 kHz 或 10 kHz 运算。
如果目标是统一预训练，一个很实用的工程折衷是： **统一存两套视图** ——`EEG_cortex@128Hz` 与 `EEG_fast@1kHz`，让 tokenizer 与 alignment 模块各取所需。ds006434 的作者已经显示，真实实验中还必须做  **clock-drift correction by resampling** ，否则 latency 估计会偏；同时对长静音段需要显式屏蔽，否则会把 offset/onset ERP 混进连续回归模型。

在音频侧，建议不要直接把原波形送入跨模态对齐器。更稳妥的做法是构建 **多分辨率音频特征栈** ：

* 语音：16 kHz 单声道；80-bin log-Mel（25 ms 窗、10 ms hop）；sub-band envelope；F0/voicing；ASR forced alignment 输出的 phoneme/word boundary；若有文本，再加词级语义嵌入。
* 音乐：22.05 kHz 或 44.1 kHz；onset strength、tempogram、HPCP/chroma、beat/downbeat 标记；必要时做 harmonic/percussive 分离。
  这样做的核心理由，是让 EEG token rate 与音频特征 rate 在 10–100 Hz 之间可对齐，而不是让 128 Hz EEG 去“追逐” 16 kHz 原波形。对于 ds004718 这类已提供每 10 ms prosody 的数据，可直接把 10 ms 注释作为事件支路监督；对于 KUL/DTU/4518754，则以 envelope / onset 为主。

在 **continuous-to-discrete tokenization** 上，我的推荐顺序是这样的。
第一优先级是 **分层残差向量量化** ：先用轻量 TCN / Conformer encoder 把 EEG 压到 25–50 Hz 的 latent，再用 2–4 层 RVQ 量化，每层 codebook 可从 256 或 512 开始。经验上，`latent_dim=128~256`、`codebook_size=256~1024`、`commitment_beta=0.1~0.25`、`EMA_decay=0.95~0.99` 是很稳的起点。
第二优先级是 **CPC/InfoNCE 风格的预测式离散化** ：先学连续 latent，再在 latent 空间做 Gumbel-VQ 或 k-means/offline product quantization。它在跨被试预训练时往往更稳定，因为你可以先学“可预测性”，再学“离散边界”。
第三优先级是 **temporal convolutional discretizer** ：直接让 1D 卷积编码器输出每个时间步的 token logits，配合 entropy regularization 与 token usage regularization。
如果你的目标是“跨数据集可迁移”的 codebook，我更建议 **先连续表征、后离散化** ，并在离散化前做 subject-adversarial / instance normalization；这样可以减少“一个 token 只代表某个被试的阻抗/参考方式”的风险。这个建议尤其适用于 KUL、DTU、4518754 与 ds004408 这类跨设备/跨设置差异明显的数据。

在 **alignment** 上，可以明确分三层方法。
低层我建议以 **TRF / ridge regression / CCA / cross-correlation** 为起点，因为它们能最快告诉你“EEG 与哪种音频特征最同步、最佳 lag 是多少”；
事件级我建议用  **DTW / Soft-DTW / constrained monotonic alignment** ，特别适合词边界、phoneme onset、beat/downbeat 这类稀疏事件；
高层语义则用  **cross-modal transformer with local monotonic attention** ，并在训练时加一个  **lag prior** （例如 0–400 ms 的因果窗），避免模型用不合理的超前信息“作弊”。
一个实用技巧是：先用 CCA/TRF 粗估每个被试的 **全局 lag** ，再把这个 lag 作为 transformer 的初始化偏置或 positional shift；这样比端到端从零学对齐要稳得多。对 ds006434 这种高精度同步数据，可以把 lag 搜索窗缩到 ±20 ms；对自然语音皮层数据，搜索窗设到 0–500 ms 更合理。

评价指标建议也分层。
对 tokenizer，本身要看  **codebook perplexity、usage entropy、dead-code ratio、跨被试 token overlap** ；若有标签，还应看  **phoneme/word/beat 的 token purity、NMI、ARI、ABX 识别率** 。
对低层对齐，看  **TRF prediction correlation、CCA canonical corr、lag estimation error、framewise retrieval R@K** 。
对事件级对齐，看  **onset F1、mean absolute latency error、DTW path cost** 。
对高层对齐，看  **word/sentence retrieval、cross-modal contrastive accuracy、speaker-attention decoding AUC** 。
真正有价值的报告方式是：在同一数据集上同时给出“低层好不好、事件级好不好、高层好不好”，而不是只报一个单一准确率。尤其在 KUL 上，如果只报 1 s 窗口的 attended-side classification accuracy 而不说明 trial split，结果没有解释力。

数据增强方面，EEG 侧建议使用： **channel dropout、small temporal jitter、band-limited Gaussian noise、frequency masking、随机参考扰动后的重参考一致化** ；音频侧建议使用： **SpecAugment、轻微 loudness perturbation、room convolution、局部 time masking** 。但要注意两条原则：
第一， **不要破坏真实跨模态同步结构** ，所以强 time-stretch 只适合自监督预训练，不适合监督对齐；
第二， **AAD 任务不要做会泄漏 attended side 的增强** ，例如只对一侧声道做固定变换。
另外，跨被试训练时最好加 **subject-balanced batching** 与  **device/domain adversarial loss** ，否则 tokenizer 很容易学到“设备指纹”。

## 推荐实验路线图

**第一阶段应先做“不会错”的低层对齐基线。**
用 **ds004408、KUL、DTU、4518754** 四个集合作为低层同步基准，统一到 `EEG_cortex@128Hz` 与 `audio_env@64Hz/128Hz`，先跑 ridge-TRF、backward envelope reconstruction、CCA 与简单双塔对比学习。目标不是追最高精度，而是先回答三件事：哪类音频表征最稳、每个被试的最佳 lag 分布如何、你设计的 tokenizer 会不会只学到设备/试次指纹。KUL 必须使用 leave-one-trial-out 或 leave-one-story-out；结果中必须同时报告 subject-wise 与 trial-wise 泛化。

**第二阶段再上“词/音素事件级对齐”。**
这里我建议用 **ds006104 + ds004718** 形成一个互补组合：前者做受控音位级验证，后者做自然语音词级验证。

* 在 ds006104 上，检验 EEG token 是否能稳定聚成 articulatory / phonemic 类别；
* 在 ds004718 上，检验同一 tokenizer 在自然语音中是否还能保留词边界和 prosody。
  若这一步失败，说明你的离散器虽能做低层同步，但并未真正编码可迁移的语言事件结构。

**第三阶段做“完整语义对齐”。**
把 **ds005345、ds004718、ds004408** 作为主语义训练/测试床：
先用 ds004408 预训练自然语音 EEG encoder；
再在 ds004718 上加词边界与 prosody 监督；
最后在 ds005345 上评估单讲者/多讲者、说话人选择与文本检索。
如果只做一个最有论文产出的实验，我会优先做这个三段式 pipeline。

**第四阶段做“高采样率与高密度扩展”。**
把 **ds006434** 与 **4518754** 作为扩展集，分别研究：

* 更高时间精度对 token 边界是否有帮助；
* 更高空间密度是否能提升 codebook 的跨被试稳定性。
  这一步尤其适合验证“同一个离散器是否需要 dataset-specific codebook”，还是可以通过 domain-adaptive encoder 共享 codebook。

**第五阶段再进入音乐。**
先用 **OpenMIIR** 做 beat/downbeat/tempo 对齐，因为它的结构标注最清楚；再用 **MUSIN-G** 做自然长音乐片段的自监督预训练；如果之后你还想做“目标乐器注意”，再去补 **MAD-EEG** 这类小样本音乐 AAD 数据。音乐方向不要一开始就追“重建波形”，而应先验证 EEG token 是否能锁定节拍、拍点、歌词/无歌词差异与乐段边界。

## 开放问题与局限

当前公开资源虽然已经足够支持一个强有力的 EEG–音频跨模态计划，但仍有几个需要在开工前手动核对的问题。第一， **个别仓库中的音频文件是否完整镜像、采样率是否在 README 中明确写出** ，尤其是 MUSIN-G。第二， **外部 Zenodo 数据集的 license 字段** ，这次抓取中对 DTU 与 255ch AAD 的许可元数据没有完整展开，正式整理数据卡前应再核对一遍仓库元数据。第三，**ds005345 的参与者数量**在当前 README 与论文摘要之间存在轻微不一致，实际建 benchmark 时应以下载后的 `participants.tsv` 为准。第四，**LPPHK 的词切分**部分依赖人工步骤，论文已明确提示这一点，所以如果你要把它作为“金标准对齐集”，建议先做一轮注释一致性抽查。

最后，如果你的最终目标是“EEG token ↔ 语音/音乐 token 的统一跨模态字典”，我最推荐的研究顺序是： **先低层同步，后事件级离散化，再高层语义耦合** 。直接从文本检索或 waveform reconstruction 起步，往往会把同步误差、试次偏差、设备差异和标签稀疏问题全部混在一起，反而难以定位瓶颈。现有公开数据已经足够支持一种更稳、更可解释、也更容易复现的路线。
